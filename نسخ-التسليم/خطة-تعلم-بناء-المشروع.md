# خطة تعلم بناء مشروع Maktab Plus من الصفر

هذه الخطة هدفها أن تتعلم كيف تبني مشروع **Maktab Plus** خطوة بخطوة، بدل الاكتفاء بتشغيل المشروع الجاهز.

المشروع الأصلي هو نظام ويب لإدارة مكاتب الخدمات، ويعتمد على:

- Next.js
- React
- TypeScript
- Tailwind CSS
- PostgreSQL
- نظام تسجيل دخول وصلاحيات
- إدارة منشآت متعددة
- عملاء وعمليات وسندات وفواتير وتقارير

## جدول الدراسة المقترح

المدة المقترحة: 10 أسابيع.

عدد أيام الدراسة: 5 أيام في الأسبوع.

الوقت اليومي: من ساعة ونصف إلى ساعتين.

| الأسبوع | الهدف | ماذا تتعلم وتبني |
|---|---|---|
| 1 | أساسيات الويب | HTML, CSS, JavaScript، وفهم الصفحة والنموذج والزر |
| 2 | React | المكونات، props، state، forms، عرض القوائم |
| 3 | Next.js | إنشاء مشروع، الصفحات، التنقل، App Router، layout |
| 4 | التصميم | Tailwind، واجهة عربية RTL، sidebar، buttons، cards، forms |
| 5 | قاعدة البيانات | PostgreSQL، الجداول، العلاقات، العملاء والعمليات |
| 6 | API | إنشاء مسارات مثل `/api/clients` و `/api/contracts` للإضافة والعرض والتعديل |
| 7 | تسجيل الدخول | users، تشفير كلمة المرور، sessions، cookies |
| 8 | الصلاحيات | أدوار المستخدمين: مالك منصة، مالك مكتب، موظف، وتحديد صلاحيات كل مستخدم |
| 9 | الفواتير والتقارير | فواتير، بنود الفاتورة، كشف حساب العميل، ملخص لوحة التحكم |
| 10 | النسخة النهائية | النسخ الاحتياطي، الإعدادات، تحسين الواجهة، تجربة التشغيل والإنتاج |

## طريقة الدراسة اليومية

| الوقت | النشاط |
|---|---|
| 20 دقيقة | شرح الفكرة ببساطة |
| 40 دقيقة | كتابة الكود خطوة بخطوة |
| 30 دقيقة | تمرين عملي صغير |
| 20 دقيقة | مراجعة وتصحيح وفهم الأخطاء |

## خطة بناء النسخة الأولى

ابدأ بنسخة صغيرة جدًا، ثم وسعها تدريجيًا:

1. إنشاء مشروع Next.js جديد.
2. بناء صفحة تسجيل دخول شكلية.
3. بناء لوحة تحكم بسيطة.
4. إضافة صفحة العملاء.
5. ربط العملاء بقاعدة PostgreSQL.
6. إضافة العمليات المالية.
7. حساب رصيد العميل.
8. إضافة السندات.
9. إضافة الفواتير.
10. إضافة المستخدمين والصلاحيات.

## الدرس الأول

ابدأ بدرس:

**إنشاء مشروع Next.js من الصفر وتشغيله**

في هذا الدرس تتعلم:

- معنى مجلد `app`.
- معنى ملف `layout`.
- معنى ملف `page`.
- كيف تعمل الصفحة الرئيسية.
- كيف تضيف صفحة جديدة.
- كيف تشغل المشروع محليًا.

## ترتيب بناء المشروع بشكل عملي

### المرحلة 1: الواجهة الأساسية

- صفحة تسجيل الدخول.
- صفحة لوحة التحكم.
- القائمة الجانبية.
- تصميم البطاقات والأزرار.
- دعم اللغة العربية واتجاه RTL.

### المرحلة 2: العملاء

- جدول العملاء.
- نموذج إضافة عميل.
- تعديل عميل.
- حذف عميل.
- منع تكرار رقم الجوال.

### المرحلة 3: العمليات المالية

- إضافة عملية مالية.
- تحديد نوع العملية.
- ربط العملية بعميل.
- عرض عمليات كل عميل.
- حساب الرصيد.

### المرحلة 4: السندات

- سند قبض.
- سند صرف.
- تحويل المبلغ إلى كتابة.
- طباعة السند.

### المرحلة 5: الفواتير

- إنشاء فاتورة.
- إضافة بنود للفاتورة.
- حساب الإجمالي.
- طباعة الفاتورة.
- تقرير فواتير اليوم.

### المرحلة 6: المستخدمون والصلاحيات

- إنشاء مستخدم.
- تسجيل الدخول.
- حفظ الجلسة في cookie.
- تحديد صلاحيات كل مستخدم.
- إخفاء الصفحات حسب الصلاحيات.

### المرحلة 7: المنشآت المتعددة

- إنشاء أكثر من منشأة.
- ربط كل عميل وعملية بمنشأة.
- منع اختلاط بيانات المنشآت.
- إدارة الاشتراك التجريبي أو النشط.

### المرحلة 8: النسخ الاحتياطي والإنتاج

- إنشاء نسخة احتياطية.
- استيراد بيانات.
- إعداد متغيرات البيئة.
- تشغيل المشروع في وضع الإنتاج.

## ملاحظات مهمة

- لا تبدأ بكل ميزات المشروع مرة واحدة.
- ابن نسخة صغيرة تعمل، ثم أضف ميزة واحدة في كل مرة.
- بعد كل ميزة، جرّبها من الواجهة ومن قاعدة البيانات.
- اكتب أسماء واضحة للملفات والمكونات.
- افهم الفكرة قبل نسخ الكود.
